# 一个月烧完 200 亿 AI Token，能做出什么样的产品？

最近拿到了一笔 OpenAI 的 AI credit，我决定做一个有点极端的实验：
在一个月内，烧掉 200 亿 token，回答两个问题：

1. AI 除了工程之外，在其他运营任务上到底应该怎么用？
2. 用 AI 做出的“最有价值的产品”，长什么样？

这件事的起点，是我最近一直在反复想的一个问题。
过去一段时间，我一直在写一个关于 Agent 的产品 thesis。
其中一个核心判断是：
未来不管是信息检索还是任务执行，生产的主体很可能会从“人”逐渐转向“Agent”。

如果这件事真的发生，变化可能不只是效率提升。
它会连带改变很多我们已经习以为常的产品形态。

比如广告。
如果未来真正“消费信息”的不再是人，而是 Agent，
那广告到底是给谁看的？
现在围绕点击率和转化路径设计的一整套逻辑，
可能都需要被重新定义。

再比如界面。
随着多模态 Agent 和手机侧 Agent 的发展，
很多任务已经可以直接让 Agent 完成。
在这种情况下，复杂 GUI 反而会变成负担：
人未必看得懂，操作成本也高。
如果任务可以被清晰表达，那直接让 Agent 执行，体验反而更好。

类似变化已经在工具层开始发生。
现在很多开发工具，比如数据库、后端服务、邮件系统，
只要做得足够标准化，Agent 在执行任务时就可以直接调用。
像 Supabase、Resend 这一类服务，
正在逐渐变成“Agent 默认可用的基础设施”。

但这可能只是第一层变化。
再往下一层，会不会有更多原本“为人设计”的产品，
需要被重新做一遍？

比如 Yelp 这种，本质是给人看的信息浏览与决策工具。
如果未来是 Agent 帮你选餐厅，产品形态会不会完全不一样？
再比如大量 SaaS 工具，今天的交互逻辑和信息结构，
很多都是围绕“人如何理解和操作”设计的。
如果使用者变成 Agent，这一套是不是也要重构？

这背后有个很现实的机会：
有些公司很难快速完成这种转变，
而“为 Agent 设计”的产品形态，本身可能就是一条新路径。
甚至有些服务，可能需要同时存在两种版本：
一套给人用，一套给 Agent 用。

从组织层面看，我最近越来越在意一个更底层的问题：
如果生产主体真的从人转向 Agent，
企业之间的竞争力会变成什么？

我现在的直觉是：
未来公司之间比的，可能不再是传统意义上的“人效”，
而是一个更直接的指标——Token ROI（Token 使用效率）。

同样消耗 1000 个 token，
有的团队只是生成一段内容，
有的团队却能把它变成一个功能、一个增长点，甚至直接转化成收入。

Token 成本在不断下降，
但真正拉开差距的，是你能不能用这些计算创造出实际价值。

换句话说，消耗多少 token 并不重要，
重要的是这些 token 最终换来了什么。

从这个角度看，一个 AI 原生团队可能可以用两个维度衡量：
1）总共消耗了多少 token；
2）这些 token 的产出效率。
前者决定上限，后者决定质量。

我之前写代码时也有类似感受：
有些问题需要反复迭代、消耗大量计算，但效果一般；
但有些地方只是几行代码，就能带来明显结果。
营销、SEO、获客其实也是一样，
本质都是用很小成本撬动更大回报。

现在像 OpenClaw 这样的工具，
已经可以同时覆盖研发、产品，甚至一部分市场工作。
如果这套能力成立，
更彻底的方向可能不是“用 AI 提效”，
而是让一个团队从一开始就是 AI 原生的。

也就是说，不只是“用 AI”，
而是围绕“如何更高效地消耗 token 并产生价值”来设计整个组织。

所以这次“200 亿 token”实验，
对我来说不只是一个烧钱挑战，
更像是在验证一件事：
当你不再刻意节省 token，
而是反过来放大计算、让 Agent 去处理更多不确定性时，
产品会不会走向一种完全不同的形态。

接下来一个月，我会持续记录这个过程：
这些 token 被用在了哪里，
哪些是有效的，哪些是浪费的，
以及在这种前提下，能不能真的跑出一个有价值的产品。

如果你也在做 Agent，或者在思考 AI 产品这件事，
我也很好奇：
如果调用 AI 的成本几乎为零，
你会怎么设计你的产品？
你觉得目前最容易产生实际收益的方向是什么？
